Introduction

This file is created when AP_SETUP_DIR/ap-setup.sh is used to deploy the Relay Server. This quick reference allows users with minimal Relay Server and Web Server experience to deploy Relay Server on an Apache server. It is recommended that users read the Introduction to the Relay Server documentation before using this guide. This guide will be re-created if ap-setup.sh is run again.

Editing and Deploying Relay Server Configuration

The initial Relay Server configuration file, RS_CONFIG, generated by the setup script is a basic configuration file with only one sample back-end farm and back-end server for a SimpleTestApp using the local Apache server as the back end. It contains only this single Relay Server. To allow it to work with your own back-end farms and servers, this Relay Server needs to be further configured before it can provide the relay service. There are two kinds of edit and deploy options available:

1.       GUI based administration: Sybase Central is a GUI tool that contains a Relay Server plug-in for remotely importing, editing and distributing configuration files for a farm of Relay Servers.

2.       Local edit and deploy: Back up and then edit RS_CONFIG using a local text editor. A sample configuration is available at SA_AP_BIN/rs.config.sample. Online documentation for the Relay Server Configuration File is also available. After editing the configuration file, the following command can be used to deploy the new configuration to an already running Relay Server without service interruption. Before you run this command, make sure LD_LIBRARY_PATH is set to include the SQL Anywhere libraries directory.

SA_AP_BIN/rshost -u -f "RS_CONFIG"

Starting and Stopping Relay Server

The Relay Server is a composite of two main components, Relay Start Management Service and Apache webserver, that can be manipulated independently but are both required to deliver any Relay Server functionality.

1. Relay Server State Manager Service

The Relay Server State Manager Service loads and provides Relay Server configuration as well as maintaining state information for inter-process communication between Relay Server components. A service called SimpleTestApp_rst has been set up by rs-setup.bat using the dbsvc utility.

The following is a list of common commands for Relay Server State Manager Service operations:

·         To start:               dbsvc -u SimpleTestApp_rst

·         To stop:                dbsvc -x SimpleTestApp_rst

·         To review:           dbsvc -g SimpleTestApp_rst

2. Apache webserver

The Apache webserver is required for publishing the Relay Server components to Sybase Central Relay Server plug-in, SQL Anywhere Monitor for Relay Server, Relay Server Outbound Enabler and various clients of the various back-end services.

The following is a list of basic commands for manipulating the Apache webserver

·         To start:               <apache_dir>/bin/apachectl start

·         To stop:                <apache_dir>/bin/apachectl stop

·         To restart:           <apache_dir>/bin/apachectl restart

Providing Back-end Service

In general, a Relay Server Outbound Enabler needs to be started on the back-end server machine before the client can access the back end via the Relay Server. For quick testing and demo purpose, rs-test-setup.sh has set up a sample RSOE service on this machine. Use the following commands to manipulate the demo RSOE service:

·         To start:               dbsvc -u SimpleTestApp_oe

·         To stop:                dbsvc -x SimpleTestApp_oe

·         To review:           dbsvc -g SimpleTestApp_oe

Client accessing Back-end Service via Relay Server

The general format of the URL that a client can use is:

APACHE_URL/cli/iarelayserver/<backend_farm>[/<app_specific_path>]

The following URL points to the SimpleTestApp, which can be used as a quick test of your setup:

APACHE_URL/cli/iarelayserver/SimpleTestApp-farm/SimpleTestApp.htm

Monitoring Relay Server

The SQL Anywhere Monitor for Relay Server provides a full-featured data collection, monitoring and alert notification facility. If you are interested only in the general service availability of the back-end services via Relay Server, there are a various Relay Server Status Pages available to provide status information at different levels for different audiences through different extensions. Example status pages are listed in the Troubleshooting Resources section below.
