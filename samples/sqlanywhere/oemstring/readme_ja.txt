		   データベースからの OEM 文字列抽出のサンプル

目的
----
このフォルダーには、データベースファイルヘッダーからの OEM 文字列抽出
方法を示す 2 つのサンプルが含まれています。

必要条件
--------
PERL サンプルは、以下を前提としています。
 - 使用するコンピューターに perl がインストール済みである事。

C サンプルは、オペレーティングシステムの種類にかかわらず、以下を前提としています。
 - 環境変数 SQLANY16 に SA フォルダーが設定されている。
   これは、SQL Anywhere のインストール時に設定されます。
Windows の場合、さらに以下も前提としています。
 - makefile を使用するために、Visual Studio .NET 2005 または 
   それ以上 がインストール済みである。
 - 環境変数 VCINSTALLDIR が設定済みである。これは、Visual Studio の
   インストール時に設定されますが、Visual Studio によってインストールされる 
   vcvars32.bat ファイルを使用して設定することもできます。

ソースコードは、他のコンパイラでもコンパイルできます。

UNIX 専用のビルド手順については、build.sh ファイルを参照してください。

処理手順
--------
build.bat (UNIX は build.sh) を実行すると、dboem.exe (UNIX は dboem)
バイナリをコンパイルします。

データベースに OEM 文字列がまだ設定されていない場合、データベースを起
動して以下のコマンドを実行します。
	dbisql set option public.oem_string = 'This is a sample OEM string.'
その後、データベースをシャットダウンします。


以下のいずれかのコマンドを実行します。
	Windows - .\dboem.exe <name-of-a-database-file>
	UNIX    - ./dboem <name-of-a-database-file>
または
	Windows - perl .\dboem.pl <name-of-a-database-file>
        UNIX    - perl ./dboem.pl <name-of-a-database-file>

これらのコマンドは、どちらもデータベース内に現在設定されている OEM 文
字列を出力します。

perl スクリプトは、デフォルトでは OEM 文字列を 16 進法で出力します。
perl スクリプトに単一のテキスト行を出力させるには、以下のコマンドを使
用します。
	perl .\dboem.pl -t <データベースファイル名>


clean.bat (UNIX は clean.sh) を実行すると、生成され
たすべてのファイルを削除します。
